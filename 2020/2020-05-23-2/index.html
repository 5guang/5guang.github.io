vuecli多页面配置踩坑笔记

Written by 5guang with ♥ on May 23, 2020 in

因为新入职的公司项目同时有 pc 和移动端的，逻辑基本上是一样，但是 ui 不同，之前是两套代码，有新需求，得两套都得写一遍。所以我使用 vue 做一个多页面应用，说是多页面，其实只有两个页面，一个 h5 的，一个 pc 的，各自做成单页面应用。下面说一下用 vuecli 配置多页面中遇到的坑：

坑点 1

vuecli 配置多页面的入口是一个 pages 属性, 下面是 vuecli 官网给出的说明（vuecli官网链接）：

// vue.config.js
module.exports = {
  pages: {
    index: {
      // page 的入口
      entry: 'src/index/main.js',
      // 模板来源
      template: 'public/index.html',
      // 在 dist/index.html 的输出
      filename: 'index.html',
      // 当使用 title 选项时，
      // template 中的 title 标签需要是 <title><%= htmlWebpackPlugin.options.title %></title>
      title: 'Index Page',
      // 在这个页面中包含的块，默认情况下会包含
      // 提取出来的通用 chunk 和 vendor chunk。
      chunks: ['chunk-vendors', 'chunk-common', 'index'],
    },
    // 当使用只有入口的字符串格式时，
    // 模板会被推导为 `public/subpage.html`
    // 并且如果找不到的话，就回退到 `public/index.html`。
    // 输出文件名会被推导为 `subpage.html`。
    subpage: 'src/subpage/main.js',
  },
};

坑点就是在 chunks，你的入口的名称一定要和 chunks 的一致，下面看看我之前的配置：

// vue.config.js
const pages = {
  wap: {
    entry: 'src/pages/wap/main.ts',
    template: 'src/pages/wap/index.html',
    filename: 'wap.html',
    title: 'wap',
    chunks: ['chunk-vendors', 'chunk-common', 'index'],
  },
  web: {
    entry: 'src/pages/web/main.ts',
    template: 'src/pages/web/index.html',
    title: 'web',
    filename: 'web.html',
    chunks: ['chunk-vendors', 'chunk-common', 'index'],
  },
};

修改之后：

// vue.config.js
const pages = {
  wap: {
    entry: 'src/pages/wap/main.ts',
    template: 'src/pages/wap/index.html',
    filename: 'wap.html',
    title: 'wap',
    chunks: ['chunk-vendors', 'chunk-common', 'wap'],
  },
  web: {
    entry: 'src/pages/web/main.ts',
    template: 'src/pages/web/index.html',
    title: 'web',
    filename: 'web.html',
    chunks: ['chunk-vendors', 'chunk-common', 'web'],
  },
};

从上面看出，其实就是入口的属性名要和 chunks 的名称一致。

所以我们在复制粘贴的时候不能粘完就完事了，虽然我们都是cv工程师 -.-

坑点 2

因为我们的项目是通过 ssi 注入到 html 的碎片化形式，所以必须要保留注释。
类似这样：

<div style="display: none;">
  <!--#include virtual="<%= VUE_APP_VIRTUAL %>/6455_100990986.html" -->
  <div class="i-nav-wrap">
    <!--#include virtual="<%= VUE_APP_VIRTUAL %>/6447_100990986.html" -->
  </div>
  <!--#include virtual="<%= VUE_APP_VIRTUAL %>/6450_100990986.html" -->
</div>

但是 webpack 在打包的时候会默认去掉注释，这样就导致无法通过 ssi 注入到 html。我一开始也没觉得有啥难，想通过 webpack 配置一下就好了，万万没想到， 用了我一上午的时间。。。

刚开始我是这样配置的

// vue.config.js
module.exports = {
  chainWebpack: (config) => {
    config.plugin('html').tap((args) => {
      if (process.env.NODE_ENV === 'production') {
        args[0].minify.removeComments = false;
      }
      return args;
    });
  },
};

然后开始调试，结果一打包就报错:

去vuecli官网看文档，文档上也没说明，后面想通过查看vue的webpack原始配置看看，最后发现了原因，

提示 使用 vue-cli-service inspect 来查看一个 Vue CLI 3 项目的 webpack 配置信息（包括：development、production）

  1. –mode 指定环境模式 (默认值：development)
  2. 运行命令，在终端输出：
    • 开发环境： npx vue-cli-service inspect --mode development
    • 生产环境：npx vue-cli-service inspect --mode production
  3. 运行命令，将输出导入到 js 文件：
    • 开发环境：npx vue-cli-service inspect --mode development >> webpack.config.development.js
    • 生产环境：npx vue-cli-service inspect --mode production >> webpack.config.production.js
  4. 在产生的 js 文件开头，添加：module.exports =，然后格式化即可查看。
  5. 官方网址：vuecli官网链接

下面是vuecli的webpack的文件

 // webpack.config.development.js
 /* config.plugin('html-wap') */
    new HtmlWebpackPlugin(
      {
        title: 'wap',
        templateParameters() { /* omitted long function */ },
        chunks: [
          'chunk-vendors',
          'chunk-common',
          'wap',
        ],
        template: 'src/pages/wap/index.html',
        filename: 'wap.html',
      },
    ),
    /* config.plugin('html-web') */
    new HtmlWebpackPlugin(
      {
        title: 'web',
        templateParameters() { /* omitted long function */ },
        chunks: [
          'chunk-vendors',
          'chunk-common',
          'web',
        ],
        template: 'src/pages/web/index.html',
        filename: 'web.html',
      },
    ),

从注释中可以看到多页面的HtmlWebpackPlugin插件配置修改不是通过 config.plugin('html') 来配置的，而是通过 config.plugin('html-web') 你配置的入口名来修改（其他配置的修改也是这样的），所以修改一下之前的配置：

// vue.config.js
module.exports = {
  chainWebpack: (config) => {
    config.plugin('html-wap').tap((args) => {
      if (process.env.NODE_ENV === 'production') {
        args[0].minify.removeComments = false;
      }
      return args;
    });
  },
};

ok，这样就可以保留注释了！

坑点3

因为我们的项目其实是使用的多页面的单页面应用，所以还得给每个单页面配置路由，我们的路由用的是history模式，开发模式下访问html没问题，主要是写好路由以后访问路由就有问题了，会出现：

We’re sorry but wap doesn’t work properly without JavaScript enabled. Please enable it to continue.

去vuecli官网查看（链接），通过在webpack官网(链接)找到了，需要在devserve中配置一下 historyApiFallback


// vue.config.js
module.exports = {
  devServer: {
    open: true,
    port: 8080,
    https: false,
    disableHostCheck: true,
    overlay: {
      warnings: true,
      errors: true
    },
    historyApiFallback: {
      verbose: true,
      rewrites: [
        { from: /^\/wap\/.*$/, to: "/wap.html" },
        { from: /^\/web\/.*$/, to: "/web.html" }
      ]
    }
  },
};

这样就ok了，说明一下 其中的 disableHostCheck: true, 一定要打开。

devServer.disableHostCheck配置项用于配置是否关闭用于 DNS 重绑定的 HTTP 请求的 HOST 检查。 DevServer 默认只接受来自本地的请求，关闭后可以接受来自任何 HOST 的请求。 它通常用于搭配–host 0.0.0.0使用，因为你想要其它设备访问你本地的服务，但访问时是直接通过 IP 地址访问而不是 HOST 访问，所以需要关闭 HOST 检查。
